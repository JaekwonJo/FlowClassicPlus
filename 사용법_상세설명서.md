# Flow Classic Plus 상세 사용 설명서

이 문서는 `Flow Classic Plus`를 실제로 쓰는 기준으로 다시 정리한 설명서입니다.

## 1) 먼저 설치
1. `1_필수라이브러리_설치.bat` 실행
2. 설치가 끝나면 `2_오토_프로그램_실행.bat` 실행

## 2) 지금 실제로 잘 되는 기능
- 프롬프트 자동화
- S001~S### 자동 반복
- 다운로드 자동화
- 프롬프트 생성 옵션 자동찾기/테스트
- S 생성 옵션 자동찾기/테스트

## 3) 가장 먼저 알아둘 것
### `모드`
- 이번 실행에서 최종적으로 이미지로 만들지, 영상으로 만들지 정하는 값입니다.

### `기본값 이미지 맞춤 완료` / `기본값 동영상 맞춤 완료`
- 자동화 시작 전에 사용자가 봇 작업창 상태를 한 번 맞춰줬다는 확인 버튼입니다.
- 쉽게 말해:
  - 기본값 = 출발점
  - 모드 = 이번 작업 목표

## 4) 프롬프트 자동화 사용 순서
1. 메인 메뉴에서 `프롬프트 자동화`
2. 시작 URL 확인
3. 필요하면 `🤖 봇 작업창 열기`
4. 봇 작업창에서 시작 상태를 `Image`로 맞춤
5. 앱에서 `기본값 이미지 맞춤 완료`
6. 모드가 이미지면 그대로 시작
7. 모드가 영상이면 시작 전에 영상으로 전환 후 진행
8. `프롬프트 자동화 시작`

## 5) S자동화 사용 순서
1. 메인 메뉴에서 `S001 자동화`
2. 필요하면 `🤖 봇 작업창 열기`
3. 봇 작업창에서 시작 상태를 `Video`로 맞춤
4. 앱에서 `기본값 동영상 맞춤 완료`
5. 시작번호/끝번호 확인
6. `S반복 자동화 시작`

## 6) 다운로드 자동화
- 기존 클래식 흐름과 거의 동일하게 사용
- 현재 사용자 실사용 기준으로 정상 동작 확인

## 7) 생성 옵션 자동찾기/테스트
### 프롬프트 자동화용
- 프롬프트 작업에서 쓰는 생성 옵션 기억용

### S자동화용
- S자동화에서 쓰는 생성 옵션 기억용

둘은 서로 따로 저장됩니다.

## 8) 타이핑 속도
- 슬라이더로 `x1 ~ x20` 조절 가능
- 고정 속도가 아니라 매번 조금씩 달라집니다
- 글자 속도, 행동 속도, 생각 시간이 함께 빨라지거나 느려집니다

## 9) 로그는 어디에 있나요?
- 행동 로그: `logs/action_trace_날짜_시간.log`
- 세션 리포트: `logs/session_report_날짜_시간.json`
- 생성 옵션 자동찾기/테스트 로그도 `logs` 폴더에 저장됩니다
- 앱 안의 `로그 및 미리보기 창 열기`에서도 바로 확인할 수 있습니다

## 10) 지금 바로 남아 있는 작업

- 폰트/가독성 개선
- 반응형 레이아웃
- 노트북용 브라우저 작업창 크기 조정
- 바탕화면 바로가기 정리
- 이어달리기 기능은 맨 마지막
