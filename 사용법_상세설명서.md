# Playwright 자동화 봇 상세 사용 설명서 (초보자용)

이 문서는 코딩을 몰라도 바로 쓸 수 있게, 가장 쉬운 말로 설명합니다.

## 1) 먼저 꼭 설치
1. `1_필수라이브러리_설치.bat` 더블클릭
2. 설치가 끝나면 `2_오토_프로그램_실행.bat` 더블클릭

## 2) 화면에서 꼭 입력해야 하는 3가지
앱 왼쪽에 아래 항목이 있습니다.

1. 시작 URL
- 자동화할 사이트 주소
- 예: `https://labs.google/flow`

2. 입력창 CSS Selector
- "어디에 프롬프트를 입력할지" 알려주는 주소표 같은 것
- 예시(기본): `textarea, [contenteditable='true']`

3. 제출 버튼 CSS Selector
- "어디 버튼을 눌러서 제출할지" 알려주는 것
- 예시(기본): `button[type='submit']`

## 3) CSS Selector가 뭔가요?
웹페이지 안의 요소(입력칸/버튼)를 고르는 규칙입니다.

- `#id이름` : id로 찾기
- `.class이름` : class로 찾기
- `button[type='submit']` : 타입이 submit인 버튼 찾기
- `textarea` : 텍스트 입력 박스 찾기

쉽게 말해, "이 칸/이 버튼"을 컴퓨터에게 정확히 알려주는 표식입니다.

## 4) Selector를 찾는 가장 쉬운 방법
1. 크롬에서 자동화할 페이지 열기
2. 입력칸(또는 버튼) 위에서 마우스 우클릭 -> "검사"
3. 개발자도구에서 해당 줄 우클릭 -> Copy -> Copy selector
4. 복사한 값을 앱의 selector 칸에 붙여넣기

팁:
- 너무 길고 복잡한 selector는 자주 깨집니다.
- 가능하면 짧고 안정적인 selector가 좋습니다.

앱 내 자동 기능:
- `🔍 Selector 자동 찾기` 버튼: 후보 selector를 자동으로 찾아서 칸에 채워줍니다.
- `🧪 Selector 테스트` 버튼: 현재 입력한 selector가 실제로 보이는지 검사합니다.
- `홈 화면이면 새 프로젝트 자동 클릭`: flow 홈 주소일 때 편집 화면 진입을 자동 시도합니다.
- `새 프로젝트 버튼 selector(선택)`: 자동 클릭이 실패하면 여기 직접 selector를 넣어주세요.

## 5) 헤드리스(Headless)란?
- ON: 브라우저 창을 화면에 안 보여주고 백그라운드 실행
- OFF: 브라우저 창을 보면서 실행

처음엔 OFF(체크 해제) 권장:
- 어디서 실패하는지 눈으로 확인하기 쉽습니다.

## 6) 채널(Channel)이란?
Playwright가 어떤 브라우저 엔진으로 실행할지 정하는 옵션입니다.

- `chrome`: 설치된 Google Chrome 사용
- `msedge`: 설치된 Microsoft Edge 사용
- `chromium`: Playwright가 설치한 Chromium 사용

추천 순서:
1. chrome
2. 안 되면 msedge
3. 그래도 안 되면 chromium

## 7) 실행 순서 (진짜 쉬운 버전)
1. 설치 배치 실행 (`1_필수라이브러리_설치.bat`)
2. 실행 배치 실행 (`2_오토_프로그램_실행.bat`)
3. 시작 URL 입력
4. 입력창 selector 입력
5. 제출버튼 selector 입력
6. 프롬프트 파일(`flow/flow_prompts.txt`) 확인
7. `▶ 자동화 시작` 클릭

## 8) 프롬프트 파일 형식
- 파일: `flow/flow_prompts.txt`
- 구분자: `|||`

예시:
첫번째 문장 ||| 두번째 문장 ||| 세번째 문장

## 9) 로그는 어디에 저장되나요?
- 행동 로그: `logs/action_trace_날짜_시간.log`
- 세션 리포트: `logs/session_report_날짜_시간.json`
- UI 로그창에서도 실시간 확인 가능

## 10) 자주 생기는 문제
### 문제 A: 요소를 못 찾는다고 뜸
- selector가 틀렸을 가능성이 큼
- 개발자도구에서 selector 다시 복사

### 문제 B: 입력은 되는데 제출이 안 됨
- 제출 버튼 selector가 틀렸거나,
- 사이트가 Enter 제출만 받는 구조일 수 있음

### 문제 C: 설치 배치에서 명령어 오류
- Python PATH 문제일 가능성 큼
- Python 재설치 시 "Add Python to PATH" 체크

### 문제 D: selector 테스트는 OK인데 자동화 시작하면 실패
- 대표 증상:
  - `Cannot switch to a different thread`
  - 자동화 시작 시 새 크롬창이 또 뜸
  - 입력칸/에셋검색칸을 못 찾는다고 뜸
- 원인:
  - UI 테스트 세션과 작업 세션이 스레드 충돌한 경우
- 현재 버전 기준 해결:
  - 자동화 시작 시 작업 스레드 전용 세션으로 재시작하도록 안정화됨
  - 완전중지 시 브라우저 종료도 함께 안정화됨
- 그래도 재발하면:
  1. 앱 완전 종료 후 재실행
  2. selector 자동찾기/테스트 1회
  3. 자동화 시작
  4. 로그에서 오류 줄 확인 후 전달
