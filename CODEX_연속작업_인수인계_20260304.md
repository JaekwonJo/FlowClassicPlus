## 9) 장애 원인 기록 (재발 방지용)

### 9-1. 왜 안됐는가?
- 핵심: Playwright Sync 세션을 서로 다른 스레드(UI 테스트 스레드 vs 작업 스레드)에서 재사용하면서 greenlet 충돌 발생
- 대표 로그:
  - `Cannot switch to a different thread`
  - 자동화 시작 때 새 크롬이 갑자기 하나 더 뜨거나, 로그인 세션이 끊겨 보이는 현상
  - 입력칸/에셋검색칸 탐지 실패로 연쇄 에러

### 9-2. 어떻게 막았는가?
- 자동화 시작 시 기존 세션 정리 후 작업 스레드 전용 세션으로 시작
- 완전중지 시 작업 스레드에서 브라우저 종료 보장

### 9-3. 운영 원칙
1. 안정 기준점(`53b2fe9`) 이후에는 기능 확장 전 반드시 백업 태그/브랜치 생성
2. selector 테스트 직후 자동화에서 스레드 재사용 구조를 변경하지 말 것
3. 재발 시 먼저 스레드 충돌 로그부터 확인
