- 프롬프트 자동화는 프롬프트 안의 `@999` / `@S999` 토큰을 감지해서, 그 자리에서 레퍼런스 이미지를 첨부하고 다음 문장을 이어서 입력하도록 확장

- `@` 호출이 환경에 따라 다를 수 있어 현재는 `page Shift+2 -> locator Shift+2 -> js dispatch` 순서로 재시도하고, 상단 레퍼런스 검색창도 전용 점수 로직과 후보 dump로 따로 탐지
- 레퍼런스 검색창은 보였더라도 프롬프트 입력창에 실제 `@`가 입력되지 않으면 성공으로 처리하지 않도록 보강
- 여러 명 inline 레퍼런스 뒤에는 입력창 재클릭 없이 같은 커서 위치에서 바로 이어서 입력하게 수정
- 연속 레퍼런스(`@999,@998`) 직전에는 입력창 끝 커서 정렬을 한 번 더 수행해서 다중 첨부 안정성 보강
- 레퍼런스 첨부는 `검색어 입력 -> Enter`를 기본으로 하고, Enter 뒤에도 검색창이 남을 때만 첫 결과 클릭 폴백을 수행
- 레퍼런스 첨부 후에는 입력창 재탐색 없이 기존 프롬프트 입력창을 그대로 유지
- 레퍼런스 첨부 대기시간을 크게 줄여 사용자 체감 속도를 개선
- 프롬프트 자동화 하단에서는 `레퍼런스 첨부 TEST`만 남기고 자동찾기 버튼은 제거
- 환경 모드 상태 라벨은 시작 직후에도 저장된 현재 값 그대로 보이게 수정
- inline 레퍼런스 첨부는 이제 `@`를 직접 치지 않고, 입력창 아래 `+` 버튼을 눌러 검색창을 연 뒤 `S999` 검색과 첫 결과 카드 클릭으로 붙이도록 보강

- 레퍼런스 첨부 뒤 입력창 복귀는 기존 프롬프트 입력창 근처를 우선해 다시 찾아 상단 검색창 오탐을 줄이도록 보강
