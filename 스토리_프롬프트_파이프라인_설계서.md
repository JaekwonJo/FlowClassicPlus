## 핵심 동작 방식

### Step0~4
- 사용자가 기존처럼 직접 진행
- 캐릭터/실사/상품 태그 확정 후 Step5부터 자동화 시작

### Step5
- 선택한 장면 범위의 큰 흐름 기획만 먼저 생성
- 이 결과는 배치 전체 공통 기획안으로 저장

### Step6
- 내부적으로 `15개 통배치`를 바로 돌리지 않고 `5개씩 마이크로배치`로 쪼개서 생성
- 이유:
  - 한 번에 15개 전부 생성하면 규칙 붕괴 가능성이 커짐
  - 5개씩 생성하면 번호/태그/구도 분산 유지가 더 쉬움

### Step7
- Step6 초안을 체크리스트 기준으로 다시 검수
- 출력 형식을 기계 친화적으로 덧씌워서 `[최종프롬프트] ... [끝]` 블록을 확보
- 검수 결과가 여전히 애매하면 복구 프롬프트를 1회 더 보냄

## 검증 로직

AI 검수만 믿지 않고 코드가 먼저 확인합니다.

### 코드 1차 검수 항목
- `S### Prompt : ... |||` 형식 유지 여부
- 이미지/비디오 1:1 존재 여부
- 범위 밖 번호 섞임 여부
- 비디오 방어벽 문구 존재 여부
- 이미지 프롬프트의 `@S###` 태그 존재 여부

### Step7 뒤 최종 검수
- Step7 응답에서 `[최종프롬프트]` 블록 추출
- 다시 코드 검수
- 부족하면 Step6 초안과 병합 시도
- 그래도 실패하면 복구 프롬프트 1회 추가

## 즉시 저장 구조

세션이 시작되면 아래 폴더가 만들어집니다.

- `logs/story_prompt_pipeline/<worker_name>/<timestamp>_S031_S045/`

여기 안에:
- `validated_prompts_*.txt`
  - 검증 통과한 프롬프트가 실행 중 계속 누적 저장
- `raw/`
  - Step5, Step6, Step7, repair 원문 저장
- `reports/`
  - 코드 검수 JSON 저장
- `session_manifest.json`
  - 세션 인덱스

## 채널별 파일 교체

이 파이프라인은 똑똑즈 스토리 전용으로 고정되어 있지 않습니다.

아래 4개를 바꿔 끼우면 다른 채널에도 대응 가능합니다.
- Gems 매뉴얼
- Step 매크로 문서
- 미장센/질감 라이브러리
- 장면 분할 파일

즉:
- `똑똑즈 스토리`
- `똑똑즈 경제`
- 다른 신규 채널

모두 같은 UI에서 파일 경로만 바꿔 쓰는 구조입니다.
