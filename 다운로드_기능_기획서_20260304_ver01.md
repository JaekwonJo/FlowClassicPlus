# 다운로드 기능 기획서 (2026-03-04 Ver.01)

## 0) 목적
- 기존 완성판(프롬프트 자동화, S반복 자동화)을 절대 깨지 않게 유지하면서,
- "자동화 다음 턴 시작 직전"에 이전 결과물을 다운로드하는 기능을 추가하기 위한 사전 기획 문서.
- 현재 단계는 **영상 다운로드 기능 먼저** 설계한다. (코딩 전 기획 확정 단계)

---

## 1) 최우선 원칙 (확정)
1. 기존 기능 분리:
- `프롬프트 자동화 시작`, `S반복 자동화 시작` 기존 동작은 그대로 유지
- 다운로드 기능은 완전 분리 모듈/옵션으로 추가

2. 실패 허용:
- 다운로드 실패 시 자동화 전체 중지하지 않음
- 재시도 없이 다음 작업으로 계속 진행
- 로그에 실패 기록만 남김

3. 동적 UI 대응:
- 고정 ref 번호 재사용 방식 금지
- 매 단계에서 텍스트/aria/role 기반 재탐색

---

## 2) 기능 이름/UI (확정)
1. 옵션 섹션 이름:
- `영상 다운로드`

2. 토글 이름:
- `자동화 이후 이전 영상 다운로드 사용` (기본 OFF)

3. 품질 옵션:
- 기본값: `720P`
- 선택: `1080P`, `4K`
- `270P`는 제외

4. 별도 버튼:
- `영상 다운로드 selector 자동찾기`
- `영상 다운로드 selector 테스트`

---

## 3) 적용 대상 자동화 (확정)
아래 2개 모두에 동일 메커니즘으로 적용:
1. S반복 자동화
2. 프롬프트 자동화

---

## 4) 실행 훅(시점) (확정)
핵심 시점:
- "현재 프롬프트 입력 직전"에 "이전 결과" 다운로드 시도

예시:
1. S반복:
- S02 입력 직전 -> S01 영상 다운로드

2. 프롬프트 자동화:
- 2번째 프롬프트 입력 직전 -> 1번째 결과 영상 다운로드

---

## 5) 다운로드 동작 순서 (영상) (확정)
1. 상단 검색칸에 대상 키워드 입력
- S반복: `S01`, `S02` ...
- 프롬프트 자동화: 프롬프트 맨 앞 3글자(사용 패턴상 S01/S02/S03)

2. 결과 카드 중 최신 1개 선택

3. 카드 hover -> `...` 버튼 노출

4. `...` 클릭 -> 메뉴 열기

5. `다운로드` 항목 hover

6. 품질 메뉴에서 설정 품질 클릭 (기본 720P)

7. 다운로드 완료 감지

---

## 6) 검색/식별/파일명 정책 (확정)
1. 검색 결과가 여러 개면:
- 최신 1개만 다운로드

2. 렌더링 미완료 대기:
- 최대 60초 대기
- 60초 초과 시 실패 처리 후 다음 작업 진행

3. 파일명:
- 대상 태그명 기준 저장 (`S01`, `S02`, `S03` ...)
- 강제 리네임 적용 (원본 파일명 사용 안 함)
- 프롬프트 자동화에서 앞 3글자 추출 실패(비어있음/특수문자) 시 `P001`, `P002`... 폴백 강제
- 동일 이름 충돌은 OS/브라우저 기본 동작(자동 suffix) 허용

4. 저장 폴더:
- 앱에서 별도 지정하지 않음
- 사용자가 Chrome 다운로드 기본 폴더 설정 사용

---

## 7) Selector 테스트 동작 정책 (확정)
- `영상 다운로드 selector 테스트`는 **실제 다운로드 1회 검증**으로 확정
- 이유: 메뉴 노출만 확인하면 실사용 실패를 잡기 어려움

---

## 8) 용어 설명 (사용자 질문 반영)
1. "고정 ref 재사용 금지"란?
- 한 번 찾은 요소 번호(ref)를 계속 믿고 쓰지 않는다는 뜻
- UI가 바뀌면 번호가 바뀌어 오작동하기 때문

2. "DOM 폴백 허용"이란?
- 일반 selector 탐색 실패 시,
- 페이지 내부 구조(DOM)를 직접 읽어 입력/클릭 후보를 다시 찾는 예비 경로
- 즉 "2차 안전장치"를 뜻함

권장:
- 영상 다운로드 기능에도 DOM 폴백을 예비 경로로 유지하는 것이 안정적

---

## 9) 지금까지 오류가 났던 지점 (분석 기록)
1. 스레드 충돌:
- `Cannot switch to a different thread`
- UI 테스트 세션과 작업 스레드 세션 재사용 충돌이 핵심 원인

2. 동적 UI 재할당:
- 고정 ref 의존 시 입력칸/메뉴 대상이 바뀌는 문제

3. 결과:
- selector 테스트 OK인데 실제 실행 FAIL
- 새 창 재실행/로그인 세션 분리/입력칸 미탐지 연쇄 발생

---

## 10) 구현 순서 (코딩 시작 시)
1. 영상 다운로드 설정 UI 추가 (분리 영역)
2. 영상 selector 자동찾기/테스트 구현
3. 자동화 훅 연결 (이전 결과 다운로드)
4. 실패 무중단 정책 연결
5. 60초 대기 및 최신 1개 선택 로직 연결
6. 로그 상세화
7. 실환경 검증 (S반복 -> 프롬프트 자동화 순서)

## 12) 사용자 확정 답변 반영 체크리스트 (2026-03-04)
1. 다운로드 실패 시:
- 즉시 다음 작업으로 진행 (중지/재시도 없음)

2. 최신 기준:
- 검색 결과 화면 첫 카드(정렬 기준 그대로)를 최신 1개로 사용

3. 파일명:
- `S01`, `S02`, `S03` 형태 강제 리네임

4. 프롬프트 자동화 파일명 추출 실패:
- `P001` 형태 폴백 이름 강제

5. 렌더링 대기:
- 최대 60초
- 초과 시 `태그 + 실패이유` 로그 남기고 다음 프롬프트로 즉시 진행

## 13) 기획 변경사항 (2026-03-05)
1. "프롬프트 입력 직후 다운로드" 방식은 폐기
2. 새 방식은 "결과물 전체 순차 다운로드" 전용 모듈
3. 기존 프롬프트/S반복 자동화는 그대로 유지
4. 새 실행 버튼:
- `다운로드 자동화 시작`
5. 이미지/영상 selector 도구 분리:
- 이미지 자동찾기/테스트
- 영상 자동찾기/테스트
6. 품질 정책:
- 영상 기본 `1080P` (선택 `720P`, `4K`)
- 이미지 기본 `4K` (선택 `1K`, `2K`)
7. 파일명은 사이트 원본 유지(강제 리네임 미사용)
8. 메타데이터 리포트로 태그별 성공/실패를 기록
