# 유료화 기획 초안 (2026-03-05)

## 1) 목표
- 프로그램을 월 구독형 서비스로 전환
- 요금제별 기능 제한을 명확히 분리

## 2) 요금제 구조
1. Basic (기본 요금제)
- 프롬프트 자동화
- S반복 자동화
- 기본 로그/리포트
- 다운로드 자동화 기능 미제공

2. Pro (프로 요금제)
- Basic 전체 기능 포함
- 이미지/영상 다운로드 자동화 제공
- 고급 로그/리포트(다운로드 메타데이터) 제공

## 3) 핵심 요구사항
1. 로그인/회원가입
- 이메일 기반 가입/로그인
- 비밀번호 재설정

2. 구독/결제
- 월 구독제
- 결제 성공/실패/해지 상태 동기화

3. 플랜 기반 기능 게이트
- 로그인 후 플랜 확인
- Basic/Pro에 따라 버튼/기능 활성화 분기

## 4) 권장 기술 방향
1. 백엔드
- 인증 API (회원가입/로그인/토큰 갱신)
- 구독 상태 API (Basic/Pro/만료)

2. 결제
- Stripe 월 구독 연동(권장)
- 웹훅으로 결제 상태 반영

3. 클라이언트 앱
- 실행 시 토큰/플랜 체크
- 플랜 불일치 시 기능 잠금

## 5) 개발 단계 제안
1단계: 기초 인프라
- 계정 시스템 + 로그인 UI
- 로컬 토큰 저장/갱신

2단계: 결제 연동
- 월 구독 결제 생성
- 웹훅 상태 반영

3단계: 기능 게이트
- Basic: 자동화만
- Pro: 자동화 + 다운로드

4단계: 운영/보안
- 기기 제한 정책
- 비정상 사용 탐지/차단
- 로그 감사

## 6) 정책 초안
1. Basic
- 다운로드 자동화 버튼 비활성

2. Pro
- 다운로드 자동화 전체 활성

3. 공통
- 약관/개인정보 처리방침 동의 필수

## 7) UX/UI 방향
- 초보자도 3단계 이내로 실행 가능하게 단순화
- 상태 배지(로그인/플랜/만료일) 상단 고정
- 결제/업그레이드 진입 버튼 명확화

## 8) 리스크
1. 로컬만으로는 라이선스 우회 방어가 약함
2. 결제/환불/세금 처리 정책 필요
3. 계정 탈취 대비(보안) 필요

## 9) 다음 액션
1. 요금제 가격/정책 확정
2. 인증/결제 백엔드 MVP 설계
3. 앱 내 로그인 화면 와이어프레임 제작
