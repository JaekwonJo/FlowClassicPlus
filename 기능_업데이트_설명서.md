## 대상 기능
- 안정화 기준점 고정 (프롬프트 자동화 + S반복 자동화 둘 다 동작)

- 장애 원인 기록

## 변경 전 문제 (이번 장애 핵심)
- selector 테스트는 통과했는데 실제 자동화 시작 시 실패
- S반복 Step2에서 아래 오류가 발생:
  - `Cannot switch to a different thread`
- 증상:
  - 새 크롬창이 갑자기 하나 더 뜸
  - 로그인 안 된 창/빈 세션으로 열림
  - 프롬프트 입력칸/에셋 검색칸 탐지 실패
  - 완전중지 시 브라우저 종료가 불안정

## 원인
- Playwright Sync 세션을
  - UI 스레드(selector 테스트)
  - 작업 스레드(실자동화)
  에서 섞어 재사용하면서 greenlet 스레드 충돌이 발생.

## 수정/기준점
- `flow/flow_auto_v2.py`
  - 자동화 시작 시 세션 정리 후 작업 스레드에서 새 세션 시작
  - 완전중지 시 작업 스레드에서 브라우저 종료 보장
- 안정 커밋(기준점): `53b2fe9`

## typing 속도 개선 (요청 반영)
- `flow/human_behavior_v2.py`
  - 문자 입력 딜레이를 실사용 가능한 범위로 단축
  - 과도한 문장 중간 장기 멈춤(수초~수십초) 크게 완화
  - 오타 시뮬레이션 딜레이도 단축
- 결과:
  - typing 모드가 기존 대비 훨씬 빠르게 동작
  - mixed 모드에서도 typing 분기 속도 개선

## 확인 방법
1. `🔍 Selector 자동찾기` + `🧪 Selector 테스트` 수행
2. `🚀 프롬프트 자동화 시작` 확인
3. `🚀 S반복 자동화 시작` 확인
4. `⏹ 완전중지` 시 브라우저 종료 확인
5. typing 모드로 프롬프트 1개 입력 속도 확인
